# API 510/570/653 Inspector
**ID**: `apiInspectors` - **Discipline**: Energy - **Level**: Career

## Course Aim
The API individual certification exams for pressure vessels (510), piping (570), and aboveground storage tanks (653) reward one skill above all others: finding the controlling rule fast and applying it without second-guessing. Each exam is split into a closed-book portion that tests what you must know cold and an open-book portion that tests whether you can navigate the code, ASME references, and damage-mechanism documents under the clock. The hardest part is rarely the engineering. It is knowing which document governs, which edition is in effect for your exam, and where the answer lives so you are not flipping through 600 pages with a calculator running.

This course treats the three certifications as one integrated discipline, because in a real plant the same inspector reasons across vessels, piping, and tanks using the same logic: establish scope and jurisdiction, characterize the damage mechanism, gather thickness and design data, compute corrosion rate and remaining life, set the next inspection interval, and decide whether a finding needs repair, rerating, or a fitness-for-service assessment. We drill the calculations that appear on every exam — t-min, corrosion rate, remaining life, MAWP, joint efficiency — until setup and units are automatic, then layer in the judgment that separates a passing inspector from a defensible one.

You will finish able to read a damaged-equipment scenario, name the governing code section, perform the supporting calculation, and write a recommendation a Plant Engineer and a jurisdiction would both accept. That is the standard, and it is also exactly what the exam measures.

## Course Design Notes
The three exams share a common architecture. Each pairs the inspection code (API 510, 570, or 653) with its ASME construction reference (Section VIII Div 1, B31.3, and ASME tank-relevant standards), the NDE reference (ASME Section V), the welding reference (ASME Section IX), and a shared damage-mechanism foundation in API RP 571 plus, for RBI questions, API RP 580/581. API 577 (welding inspection) and API 579-1/ASME FFS-1 (fitness-for-service) round out the body of knowledge. Because the exam is partly open book against a specific Effectivity Sheet of publication editions, this syllabus teaches you to work from the *current edition you are assigned* rather than memorizing edition-specific numbers — always confirm your exam's effectivity sheet, since editions and addenda change.

Chapters move from scope and navigation, through the engineering and damage mechanisms common to all three, into one chapter each for 510, 570, and 653 where the codes genuinely differ, and close with fitness-for-service, recordkeeping, and timed exam strategy. Every chapter includes Common traps drawn from how candidates actually lose points: misreading whether a change is a repair or an alteration, applying the wrong t-min basis, confusing inspection intervals with the maximum-period rules, and trusting memory over the open-book reference. Treat the calculations as muscle memory and the code lookups as a sport you practice for speed.

## Chapter 1: Inspector Role, Code Jurisdiction, and Reference Navigation
- **Core questions**: Which code governs this equipment, and where does its jurisdiction begin and end? Who is the owner-user, and what does the authorized inspector actually decide versus recommend? Which referenced document controls the next step?
- **Key concepts**: authorized inspector vs. owner-user vs. engineer roles; scope boundaries (where API 570 piping ends and 510 vessel scope or 653 tank scope begins); code hierarchy (inspection code → ASME construction code → Section V NDE → Section IX welding); the Effectivity Sheet and why edition/addenda matter; closed-book vs. open-book exam structure.
- **Applied skills**: Build a tabbed reference map across API 510/570/653, ASME VIII/B31.3/V/IX, and RP 571/579/580/581; locate a governing clause in under a minute; classify a scenario by which certification owns it.
- **Common traps**: Treating the inspection code as self-contained and forgetting it points to ASME for construction and NDE rules. Mixing up jurisdictional boundaries — e.g., piping connected to a vessel nozzle, or tank nozzle piping. Studying a code edition that does not match your exam's effectivity sheet.

## Chapter 2: Design Basis, MAWP, and the Numbers That Drive Inspection
- **Core questions**: What design data must exist before you can judge fitness for continued service? Which single variable controls the calculation in front of you? How does MAWP change when corrosion eats the wall?
- **Key concepts**: design pressure, MAWP, static head, corrosion allowance, nominal vs. minimum required thickness (t-min), joint efficiency E, allowable stress S, the thin-wall pressure formulas for shells and heads (ASME VIII), and how service conditions (temperature, cyclic, lethal) raise requirements.
- **Applied skills**: Compute t-min for a cylindrical shell and a head from the ASME formula; recompute MAWP from measured thickness; identify which input dominates the result and what data gap blocks the calculation.
- **Common traps**: Using nominal thickness where the code requires actual measured thickness. Forgetting to subtract future corrosion allowance before computing remaining life. Mishandling joint efficiency E — applying the wrong value because the radiography level was misread. Unit slips between inches, mils, and mm.

## Chapter 3: Corrosion, Cracking, and Damage Mechanisms (API RP 571)
- **Core questions**: Which damage mechanism does the evidence point to, and what operating condition is driving it? Where should the next inspection look, and which NDE method actually detects this mechanism?
- **Key concepts**: general vs. localized thinning, pitting, erosion/erosion-corrosion; environmental cracking (chloride SCC, caustic, amine, wet H2S, HIC/SOHIC); high-temperature mechanisms (creep, oxidation, high-temp H2 attack, sulfidation); CUI; mechanical fatigue and brittle fracture; the link from mechanism to inspection location and method.
- **Applied skills**: Match symptoms and process conditions to the most likely mechanism; choose UT, RT, WFMT, PT, or eddy current to suit the mechanism; flag CUI-susceptible temperature ranges and insulation conditions.
- **Common traps**: Naming "general corrosion" when the pattern is clearly localized (pitting or under-deposit). Missing CUI because the surface looks fine under intact insulation. Choosing an NDE method that cannot find the damage type — e.g., straight-beam UT for surface-breaking SCC instead of angle-beam or surface methods.

## Chapter 4: Corrosion Rate, Remaining Life, and Inspection Intervals
- **Core questions**: What is the corrosion rate, and how much life is left? When is the next inspection due, and is that interval defensible? Does RBI change the answer?
- **Key concepts**: short-term and long-term corrosion rate; remaining life = (t_actual − t_required) / corrosion rate; the half-life / fixed-period interval rules for internal, external, and on-stream inspection; thickness monitoring locations (TMLs/CMLs); when on-stream inspection substitutes for internal; RBI per API 580/581 and how it can extend or modify intervals.
- **Applied skills**: Calculate corrosion rate from two thickness readings; compute remaining life; set an interval as the lesser of the calculated half-life and the code maximum; explain when and how an RBI assessment justifies a different interval.
- **Common traps**: Forgetting the interval is the *lesser* of calculated remaining-life-based value and the code maximum period. Mixing short-term and long-term rates and picking the convenient one rather than the governing one. Assuming RBI always extends intervals — it can shorten them. Dropping units when corrosion rate is in mils/year but thickness is in inches.

## Chapter 5: API 510 — Pressure Vessels: Repairs, Alterations, Rerating, and Testing
- **Core questions**: Is this vessel fit for continued service? Does the proposed change count as a repair or an alteration? Is a pressure test required, and at what pressure?
- **Key concepts**: repair vs. alteration vs. rerating definitions and approval requirements; weld repair methods and the role of the engineer; pressure relief device inspection and overpressure protection; pressure test types (hydrostatic, pneumatic) and the test-pressure basis; the inspector's hold points.
- **Applied skills**: Classify a change as repair or alteration and route the required approvals; write a vessel inspection report with thickness evaluation, MAWP check, repair recommendation, and test rationale; verify pressure relief is adequate after a rerating.
- **Common traps**: Calling a rerating a simple repair and skipping engineer approval. Authorizing a pressure test without confirming the test pressure and metal temperature (brittle fracture risk on hydrotest). Forgetting that an alteration can invalidate the existing relief sizing.

## Chapter 6: API 570 — Piping: Circuits, CMLs, Injection Points, and Repairs
- **Core questions**: Which piping circuit carries the highest risk, and where should CMLs be placed? Is the proposed repair (including temporary repairs) acceptable? When is a pressure or leak test required?
- **Key concepts**: piping classes by service and consequence; circuitization; condition monitoring locations; high-risk features — injection points, mix points, deadlegs, soil-to-air interfaces, and small-bore connections; temporary vs. permanent repair rules; on-stream leak sealing limits; inspection intervals by piping class.
- **Applied skills**: Divide a system into circuits and rank them; place CMLs at injection points and deadlegs; calculate circuit corrosion rate and remaining life; judge whether a clamp/box repair is acceptable and for how long; update the piping records.
- **Common traps**: Treating a whole line as one circuit when service or corrosion regime changes mid-line. Under-monitoring injection points and deadlegs where localized loss concentrates. Letting a "temporary" repair persist past its allowable life without re-evaluation. Wrong inspection interval because the piping class was misjudged.

## Chapter 7: API 653 — Tanks: Shell, Bottom, Roof, Settlement, and Out-of-Service Rules
- **Core questions**: Does this tank need an out-of-service (internal) inspection now, or does in-service inspection suffice? What settlement pattern actually threatens integrity? Which repair and reconstruction rules apply?
- **Key concepts**: shell, bottom, and roof inspection scope; minimum thickness rules for shell courses and bottom plates; out-of-service vs. in-service inspection drivers and the bottom-integrity link to internal inspection intervals; settlement types (uniform, planar tilt, edge, localized) and their evaluation; brittle fracture considerations; repair/alteration/reconstruction per API 653 and its weld and material rules.
- **Applied skills**: Determine the out-of-service inspection interval and its basis; evaluate a settlement survey against the relevant pattern criteria; assess shell-course and bottom remaining life; scope a repair and set the next inspection date.
- **Common traps**: Confusing the in-service external interval with the out-of-service internal interval (they are governed differently). Misapplying settlement criteria — using uniform-settlement tolerance for an edge or localized settlement. Overlooking bottom-side (soil-side) corrosion that the topside reading hides.

## Chapter 8: Fitness-for-Service, Records, and Timed Exam Strategy
- **Core questions**: When does a finding move from a simple thickness check to a Level 1/2/3 fitness-for-service assessment? What record actually proves compliance? Which code section answers a given question fastest?
- **Key concepts**: API 579-1/ASME FFS-1 assessment levels and common flaw types (general/local metal loss, pitting, crack-like flaws, brittle fracture, creep); rerating as an FFS outcome; required inspection, repair, and pressure-test records and retention; welding records (Section IX, API 577 awareness); open-book search tactics, question triage, and answer elimination.
- **Applied skills**: Decide whether a finding needs Level 1 screening or escalation to an engineer for Level 2/3; write a fitness-for-service decision memo; assemble a compliant records index; build a search/tab strategy that turns the open-book sections into fast points.
- **Common traps**: Attempting an FFS judgment that requires Level 3 / engineer analysis as if it were a Level 1 screen. Treating documentation as optional when the exam (and a jurisdiction) treats the record as the proof. Burning open-book time reading instead of going straight to the indexed clause. Letting one hard closed-book item eat the time budget for several easy ones.

## Capstone
Build an integrity packet for a fictional facility containing one pressure vessel (510), one piping circuit (570), and one aboveground storage tank (653), each with realistic design data, service conditions, and two sets of thickness readings taken years apart. For each asset: state the governing code and scope decision; identify the controlling damage mechanism with the operating driver; compute corrosion rate, remaining life, and (for the vessel) recomputed MAWP; set the next inspection interval with its basis; recommend repair, alteration/rerating, or fitness-for-service escalation and justify it with a cited clause; and write the test rationale where a test applies. Close the packet with a records index and a one-page FFS escalation decision showing which findings you can clear at Level 1 and which you must hand to the engineer — the same judgment the exam, and the jurisdiction, will hold you to.

## Assessment Ideas
- Code-navigation drills graded on speed, correct reference, and section-level support — timed to mirror the open-book portion.
- Calculation sets (t-min, MAWP, corrosion rate, remaining life, interval) graded on setup, units, governing-value selection, and conclusion.
- Damage-mechanism diagnosis graded on mechanism identification, operating driver, and matching the right NDE method.
- Capstone graded on scope correctness, calculation accuracy, repair/alteration/rerating judgment, records quality, and fitness-for-service escalation logic.
