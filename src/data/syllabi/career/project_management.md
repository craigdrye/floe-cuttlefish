# Project Management
**ID**: `projectManagement` · **Discipline**: Career · **Level**: Career

## Course Aim
Build practical project execution skill: define scope, align stakeholders, plan work, manage risk, control change, communicate status, and recover when the plan meets reality. This is the non-certification sibling of PMP: fewer acronyms, more useful muscle.

## Course Design Notes
The original syllabus was sensible but compact. This rewrite expands it into a workplace-ready course spanning project initiation, scope, stakeholders, schedule, risk, communication, agile/waterfall/hybrid delivery, vendor work, recovery, and a capstone project control room.

## Chapter 1: Project Thinking and Initiation
**Key concepts**: project vs operation, objective, deliverable, sponsor, stakeholder, constraints, assumptions, success criteria, charter, business case.

**Practical questions**
- What problem is this project solving?
- Who owns the outcome, and who can block the work?
- What does "done" mean in measurable terms?

**Workplace skill**
- Write a project charter with purpose, scope, success criteria, milestones, assumptions, risks, and decision rights.

## Chapter 2: Scope, Requirements, and Work Breakdown
**Key concepts**: scope statement, requirements, acceptance criteria, WBS, work package, dependency, milestone, deliverable, non-goal, change request.

**Practical questions**
- What is included, excluded, uncertain, or negotiable?
- Which requirement is really a solution disguised as a need?
- What work packages make ownership visible?

**Workplace skill**
- Build a WBS and scope boundary statement for a messy cross-functional project.

## Chapter 3: Stakeholders, Communication, and Operating Rhythm
**Key concepts**: stakeholder map, RACI, communication plan, meeting cadence, status report, decision log, action log, escalation, governance.

**Practical questions**
- Who needs to know, decide, approve, do, review, or be reassured?
- Which meetings create decisions, and which are just calendar fog?
- What status signal would change leadership behavior?

**Workplace skill**
- Design a project operating rhythm: meetings, artifacts, updates, owners, and escalation rules.

## Chapter 4: Schedule, Resources, and Critical Path
**Key concepts**: activity list, dependency, sequencing, critical path, float, bottleneck, resource contention, buffer, milestone plan, Gantt chart, Kanban board.

**Practical questions**
- Which task actually determines the finish date?
- Where are we pretending people have infinite time?
- Which delay needs resequencing, de-scoping, or escalation?

**Workplace skill**
- Build a dependency map and identify critical path, float, bottlenecks, and schedule recovery options.

## Chapter 5: Risk, Issues, and Change Control
**Key concepts**: risk register, probability, impact, trigger, owner, mitigation, contingency, issue log, root cause, change request, impact analysis.

**Practical questions**
- What might happen, what already happened, and what must be decided?
- Which risk deserves action now?
- What is the scope/schedule/cost/quality impact of this change?

**Workplace skill**
- Create a risk register and change-request packet with options, tradeoffs, and recommendation.

## Chapter 6: Delivery Methods: Predictive, Agile, and Hybrid
**Key concepts**: waterfall, agile, Scrum, Kanban, backlog, sprint, release, iteration, velocity, WIP, hybrid governance, tailoring.

**Practical questions**
- Is the work stable enough to plan deeply upfront?
- Where do we need iteration and user feedback?
- How do we keep governance without freezing learning?

**Workplace skill**
- Select a delivery approach for three project scenarios and defend the choice.

## Chapter 7: Quality, Vendors, and Implementation
**Key concepts**: quality criteria, testing, acceptance, procurement, contract type, vendor management, implementation plan, training, rollout, adoption, handover.

**Practical questions**
- What does quality mean before someone complains?
- Which vendor dependency needs a contract, SLA, or inspection point?
- Who has to change behavior after delivery?

**Workplace skill**
- Build an implementation checklist covering testing, training, communications, support, vendor handoff, and acceptance.

## Chapter 8: Status, Recovery, and Retrospectives
**Key concepts**: red/yellow/green status, variance, forecast, corrective action, decision memo, lessons learned, retrospective, after-action review.

**Practical questions**
- Is the project off track, or did the plan lie?
- What decision is needed from whom by when?
- What should future projects learn from this one?

**Workplace skill**
- Write an executive status update that distinguishes facts, risks, decisions, tradeoffs, and next actions.

## Capstone: Project Control Room
Learners manage a simulated project with shifting requirements, stakeholder pressure, vendor delay, quality issue, and timeline constraint. Final deliverables include:
- project charter
- WBS and scope statement
- stakeholder/RACI map
- communication rhythm
- schedule and critical path
- risk register
- change request
- status dashboard
- implementation checklist
- retrospective memo
