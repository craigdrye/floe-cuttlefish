# AP Computer Science
**Covers**: `col-ap-computer-science`, `col-ap-computer-science-principles` · **Discipline**: Computing · **Year**: 12 (ages 16-18)

## Course Aim
Prepare students across the two AP computer science pathways: AP Computer Science Principles for broad computing concepts and AP Computer Science A for Java programming, object-oriented design, data structures, algorithms, and exam-style code reasoning.

## Course Design Notes
College Board distinguishes AP CSA as Java-focused problem solving and object-oriented programming, while AP CSP emphasizes the big ideas of computing, algorithms, programming, data, the internet, and impact. This syllabus can serve as a combined bridge or be split into separate tracks.

## Chapter 1: Computing Systems, Data, and the Internet
- **Key concepts**: binary, data abstraction, compression, protocols, routing, DNS, cybersecurity, fault tolerance.
- **Practice questions**: How is information represented? What happens when data travels across the internet? What security threat is being described?
- **Student output**: an annotated diagram of a web request and a short binary/data representation exercise.

## Chapter 2: Algorithms and Computational Thinking
- **Key concepts**: sequencing, selection, iteration, abstraction, decomposition, simulation, algorithm efficiency, correctness.
- **Practice questions**: What steps solve the problem? Which condition controls the branch? Does this algorithm always terminate?
- **Student output**: pseudocode for a real-world process plus test cases.

## Chapter 3: Programming Foundations in Java
- **Key concepts**: primitive types, reference types, variables, expressions, operators, `String`, `Math`, input/output patterns.
- **Practice questions**: What is the value after this expression? Is this comparison using value or reference? Which method call is valid?
- **Student output**: small Java programs practicing expressions, strings, and method calls.

## Chapter 4: Control Flow and Methods
- **Key concepts**: `if`, `else`, boolean expressions, De Morgan's laws, loops, method parameters, return values, scope.
- **Practice questions**: How many times does the loop execute? What path does the condition choose? What should this method return?
- **Student output**: a Java utility class with methods, conditionals, loops, and tests.

## Chapter 5: Object-Oriented Programming
- **Key concepts**: classes, objects, constructors, instance variables, encapsulation, `this`, static members, inheritance, polymorphism.
- **Practice questions**: Which constructor runs? What method is called at runtime? What should be private?
- **Student output**: a class hierarchy with overridden methods and a tracing explanation.

## Chapter 6: Arrays, ArrayLists, and 2D Data
- **Key concepts**: arrays, `ArrayList`, traversal, indexing, mutation, enhanced for loops, nested loops, 2D arrays.
- **Practice questions**: What index is out of bounds? What happens when removing while traversing? How do row and column loops interact?
- **Student output**: data-processing methods for arrays, lists, and grids.

## Chapter 7: Searching, Sorting, Recursion, and AP Exam Reasoning
- **Key concepts**: linear search, binary search, selection sort, insertion sort, recursion, base case, tracing, AP free-response style.
- **Practice questions**: What is the precondition? What is the recursive call doing? Which sorting pass just occurred?
- **Student output**: a timed AP-style code-tracing and free-response practice packet.

## Chapter 8: Computing Impact and Create-Style Project Thinking
- **Key concepts**: computing innovation, data privacy, bias, digital divide, intellectual property, collaboration, documentation.
- **Practice questions**: Who benefits or is harmed? What data is collected? How should a program be documented for review?
- **Student output**: a small computing project proposal with purpose, algorithm, data, impact, and testing notes.

## Capstone
Choose one of two paths: a CSA path with a Java mini-project plus AP-style FRQ packet, or a CSP path with a computing innovation analysis and program project. Combined-course students complete a smaller version of both.

## Assessment Ideas
- Java tasks graded on correctness, style, encapsulation, and test coverage.
- CSP tasks graded on abstraction, algorithm explanation, impact, and responsible computing.
- Exam studio graded on tracing, FRQ clarity, and error correction.
