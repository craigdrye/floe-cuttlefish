# Web Foundations
**ID**: `webFoundations` · **Discipline**: Computing · **Stage**: High School

## Course Aim
Teach students how the web works from the page outward: semantic HTML, CSS layout basics, accessibility, forms, media, debugging, and responsible web publishing. The course emphasizes making real pages that are understandable to people, browsers, and assistive technologies.

## Course Design Notes
This syllabus draws on MDN Learn, W3C accessibility tutorials, and web.dev learning materials. It is intentionally foundation-first: students learn the difference between structure, presentation, behavior, and network delivery before chasing frameworks.

## Chapter 1: How the Web Works
- **Key concepts**: browser, server, URL, HTTP, HTML, CSS, JavaScript, request-response cycle, developer tools.
- **Practice questions**: What happens after a user enters a URL? Which technology controls content, style, or behavior? What can DevTools reveal?
- **Project output**: a diagram of a page request plus a short explanation of the HTML/CSS/JS split.

## Chapter 2: HTML Document Structure
- **Key concepts**: `doctype`, `html`, `head`, `body`, title, metadata, nesting, validation, page landmarks.
- **Practice questions**: What belongs in the head versus body? Which tag is missing? Why does nesting matter?
- **Project output**: a valid HTML page skeleton with title, metadata, and landmark regions.

## Chapter 3: Semantic Content
- **Key concepts**: headings, paragraphs, lists, links, images, tables, quotes, sections, articles, navigation.
- **Practice questions**: Which element communicates the meaning best? Is this a list, a table, or a set of paragraphs? What should the link text say?
- **Project output**: a semantic article page with headings, links, images, alt text, and a table where appropriate.

## Chapter 4: CSS Basics and Layout
- **Key concepts**: selectors, cascade, inheritance, box model, display, flexbox, grid basics, responsive sizing.
- **Practice questions**: Which rule applies? Why is the box larger than expected? How should this layout respond on a small screen?
- **Project output**: a responsive one-page site using readable typography, spacing, and at least one flexible layout.

## Chapter 5: Forms and User Input
- **Key concepts**: labels, inputs, buttons, fieldsets, validation, error messages, autocomplete, keyboard flow.
- **Practice questions**: Is the label programmatically connected? What feedback helps the user recover? Which input type fits the data?
- **Project output**: an accessible contact or signup form with labels, validation hints, and clear error states.

## Chapter 6: Accessibility From the Start
- **Key concepts**: semantic HTML, keyboard navigation, focus order, contrast, alternative text, captions, ARIA restraint.
- **Practice questions**: Can the page be used without a mouse? Does the heading structure make sense? Is ARIA replacing a native element unnecessarily?
- **Project output**: an accessibility audit of a student page with fixes and before/after notes.

## Chapter 7: Debugging and Quality
- **Key concepts**: DevTools inspection, validation, broken links, responsive testing, performance basics, file paths.
- **Practice questions**: Why is the style not applying? Which asset path is broken? What happens when the viewport changes?
- **Project output**: a debugging log showing at least five issues found, diagnosed, and fixed.

## Chapter 8: Publish a Small Website
- **Key concepts**: file organization, relative links, assets, version control basics, privacy, copyright, maintenance.
- **Practice questions**: What files must ship together? What content needs permission or attribution? What will be easy to update later?
- **Project output**: a polished multi-page mini-site with a home page, content page, form, accessible navigation, and source notes.

## Capstone
Design and publish a small informational website for a real or fictional school club, local project, or personal portfolio. Include semantic HTML, responsive CSS, accessible media, a form, and a short quality checklist.

## Assessment Ideas
- HTML exercises graded on semantic structure and validity.
- CSS layouts graded on responsiveness and readability.
- Capstone graded on accessibility, maintainability, and working navigation.
