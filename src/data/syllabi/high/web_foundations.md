# Web Foundations

## Course Aim
Students learn how web pages are structured, how HTML communicates meaning, and how simple front-end decisions affect accessibility, maintainability, and user experience.

## Core Units
- HTML document structure: `doctype`, `html`, `head`, `body`, titles, metadata, and semantic page regions.
- Text and media content: headings, paragraphs, lists, links, images, alt text, tables, and forms.
- Semantic HTML: choosing elements for meaning rather than appearance, including navigation, main content, articles, sections, and buttons.
- Accessibility basics: readable labels, keyboard-friendly controls, meaningful link text, heading order, and screen-reader-friendly structure.
- Web debugging: inspecting markup, fixing broken nesting, tracing missing attributes, and checking whether content still works without styling.
- Web literacy: how browsers request pages, how HTML, CSS, and JavaScript divide responsibilities, and how users interact with web documents.

## Assessment Targets
- Identify the correct semantic element for a page purpose.
- Explain why a markup choice improves accessibility or maintainability.
- Debug small HTML snippets with missing, misplaced, or inappropriate tags.
- Distinguish page content, visual presentation, and interactive behavior.
- Apply inclusive design habits such as useful alternative text and clear form labels.
