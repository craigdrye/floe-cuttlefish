# Intro to CS - Python
**Covers**: `col-intro-to-cs---python`, `col-computer-programming`, `col-computers-and-the-internet`
**Discipline**: Computing - **Year**: 10 (ages 14-15)

## Course Aim
Introduce students to computer science through practical Python programming, readable problem solving, and small projects that connect code to real data, files, APIs, and everyday systems. Students learn to design, test, debug, and explain programs rather than simply copy syntax.

## Course Design Notes
This course is lab-oriented and accessible to first-time programmers while still building rigorous habits: tracing, decomposition, naming, testing, and evidence-based debugging. It aligns well with CSTA high school practices, AP Computer Science Principles programming ideas, and introductory CS pathways such as CS50's computational thinking emphasis.

## Chapter 1: Computing, Python, and the Programming Loop
- **Key concepts**: program, algorithm, interpreter, expression, statement, syntax error, runtime error, IDE, terminal, repl, comments.
- **Practice questions**: What is the computer being asked to do? Which line causes the error? How can a program be broken into smaller steps?
- **Lab output**: a set of tiny Python programs that print, calculate, comment, and intentionally fix common first errors.

## Chapter 2: Variables, Types, and Input
- **Key concepts**: variable, assignment, naming, integer, float, string, Boolean, type conversion, `input`, f-string, arithmetic operators.
- **Practice questions**: What type is this value? What output follows after reassignment? Where is a cast needed?
- **Lab output**: an interactive calculator or personal budget tool that validates simple user input and formats results clearly.

## Chapter 3: Conditions and Boolean Logic
- **Key concepts**: comparison, `if`, `elif`, `else`, nested conditionals, `and`, `or`, `not`, truth table, boundary case.
- **Practice questions**: Which branch runs? What happens at the exact cutoff value? How can a complex condition be simplified?
- **Lab output**: a decision-making program such as a quiz engine, recommendation helper, or eligibility checker with multiple paths.

## Chapter 4: Loops and Program Tracing
- **Key concepts**: iteration, `for`, `while`, `range`, accumulator, counter, sentinel value, infinite loop, `break`, `continue`, trace table.
- **Practice questions**: How many times does the loop run? What value changes each pass? Why does this loop fail to terminate?
- **Lab output**: a loop-based simulation or game round that tracks score, repeats until a condition, and includes a trace explanation.

## Chapter 5: Functions and Decomposition
- **Key concepts**: function definition, parameter, argument, return value, print versus return, scope, helper function, docstring, precondition.
- **Practice questions**: What does this function return? Which variable is local? How should this problem be split into reusable parts?
- **Lab output**: a small library of tested helper functions used by a menu-driven program.

## Chapter 6: Strings, Lists, Dictionaries, and Data Patterns
- **Key concepts**: indexing, slicing, mutability, list method, traversal, nested list, dictionary key-value pair, frequency count, data cleanup.
- **Practice questions**: What element is at this index? How does the list change after mutation? Which data structure fits the task?
- **Lab output**: a text or survey analyzer that stores data in lists and dictionaries, counts patterns, and reports useful findings.

## Chapter 7: Files, APIs, Debugging, and Testing
- **Key concepts**: file path, read, write, CSV or JSON, API request, response, exception, `try`/`except`, unit test, assertion, debugging log.
- **Practice questions**: What happens if the file is missing? What shape does this JSON data have? Which test exposes the bug?
- **Lab output**: a data program that reads from a file or simple API, handles at least one error case, and includes focused tests.

## Chapter 8: Simple Algorithms and Project Studio
- **Key concepts**: linear search, min/max, sorting idea, algorithm efficiency, correctness, pseudocode, test case, code review, iteration.
- **Practice questions**: Does this algorithm always find the answer? Which cases should be tested? What tradeoff matters for this dataset?
- **Lab output**: a polished Python mini-project proposal, prototype, test plan, and peer code review notes.

## Capstone
Build a Python project that solves a real classroom, club, personal, or community problem. The project should include user input or data import, functions, at least one collection, conditionals and loops, error handling, a short test suite, and a final explanation of the algorithmic choices made.

## Assessment Ideas
- Programming labs graded on correctness, readable style, decomposition, and evidence of debugging.
- Short quizzes focused on tracing, predicting output, identifying errors, and explaining data structures.
- Capstone graded on usefulness, algorithm design, testing, documentation, and the student's ability to explain revisions.
