# Coding Basics
**ID**: `col-coding-basics` · **Discipline**: Tech & Life · **Year**: 1 (ages 5–6)

## Course Philosophy
Coding at this age is about **computational thinking**, not syntax. Children learn to break problems into steps, give precise instructions, spot errors, and think logically. Aligned with UK Computing KS1 and ISTE standards.

## Module 1: Algorithms & Sequences (30%)
- What is an algorithm? A set of step-by-step instructions
- Sequencing: put instructions in the right order
- "Program a friend": give directions (forward, turn left, turn right, stop)
- Everyday algorithms: making a sandwich, brushing teeth, getting dressed
- **Questions**: Reorder scrambled instructions, "what's the next step?", spot the missing step

## Module 2: Debugging (25%)
- What is a bug? A mistake in the instructions
- Find the error in a simple sequence
- Fix the error and explain what went wrong
- Understand that mistakes are normal and fixable
- **Questions**: "Which step is wrong?", drag-and-drop corrections, "what happens if we swap step 2 and 3?"

## Module 3: Loops & Patterns (25%)
- Repetition: "Do this 3 times" instead of writing it out 3 times
- Recognise repeated patterns as loops
- Connect to maths patterns (AB, AB, AB = a loop)
- Efficiency: fewer instructions can do the same thing
- **Questions**: "How many times does this repeat?", convert long sequences to loops, pattern-to-loop matching

## Module 4: Logic & Prediction (20%)
- If/then thinking: "If it's raining, then take an umbrella"
- Predicting outcomes of simple programs
- Binary choices: yes/no, on/off, true/false
- Introduction to the idea that computers follow instructions exactly
- **Questions**: "What will happen?", if/then scenario matching, true/false sorting

## Stretch Zone
- Simple grid-based path planning (move a character through a maze)
- Introduction to events: "When I press the button, the character jumps"
- Decomposition: breaking a big task into smaller parts
- Pixel art as a form of coding (grid coordinates + colour instructions)
