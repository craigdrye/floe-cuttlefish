# Coding Basics
**ID**: `col-coding-basics` | **Discipline**: Tech & Life | **Year**: 1 (ages 5-6)
**Frameworks**: UK Computing Key Stage 1, CSTA K-12 Computer Science Standards, ISTE Computational Thinking

## Course Aim
Build early computational thinking through play. Children learn to give clear steps, spot mistakes, repeat patterns, make choices with if/then language, move on grids, and create simple projects without needing to type code.

## Course Design Notes
For ages 5-6, coding begins away from the keyboard: bodies, arrows, cards, blocks, grids, songs, and drawings. The course treats mistakes as useful clues. Each chapter includes saying the plan aloud, trying it, noticing what happened, and fixing one small thing.

## Chapter 1: Steps in Order
- **Key concepts**: instruction, step, first, next, last, sequence, algorithm.
- **Practice activities**: put picture cards in order for washing hands, getting dressed, or making toast; give a partner three safe movement steps.
- **Teacher prompts**: What comes first? What happens if we swap these cards? Did we miss a step?
- **Work product**: a four-card everyday algorithm with drawings.

## Chapter 2: Clear Instructions
- **Key concepts**: clear, exact, forward, turn, stop, start, repeat back.
- **Practice activities**: "program a friend" around cones; use arrow cards to guide a toy; compare vague and clear directions.
- **Teacher prompts**: Could a computer understand that? How many steps forward? Which way should it turn?
- **Work product**: a short arrow-code path from start to finish.

## Chapter 3: Debugging Mistakes
- **Key concepts**: bug, debug, test, fix, try again, clue.
- **Practice activities**: find the wrong card in a sequence; test a path that bumps into a wall; fix one instruction and retest.
- **Teacher prompts**: Where did it go wrong? What clue helped us? What can we change first?
- **Work product**: a before-and-after debug strip showing the bug and the fix.

## Chapter 4: Patterns and Loops
- **Key concepts**: pattern, repeat, loop, again, number of times, efficient.
- **Practice activities**: clap-stomp patterns; replace "jump, jump, jump" with "jump 3 times"; build bead or block loops.
- **Teacher prompts**: What part repeats? How many times? Can we say it in a shorter way?
- **Work product**: a loop card with an action and a repeat number.

## Chapter 5: If/Then Choices
- **Key concepts**: if, then, choice, yes, no, true, false, condition.
- **Practice activities**: play "if I hold up red, then freeze"; sort weather cards into if/then actions; make classroom rule cards.
- **Teacher prompts**: What do we check first? What happens if the answer is no? Can one rule have two choices?
- **Work product**: an if/then picture rule, such as "If it rains, then wear boots."

## Chapter 6: Grids and Routes
- **Key concepts**: grid, square, start, finish, route, obstacle, left, right.
- **Practice activities**: move a counter on a floor grid; plan a route around blocks; draw arrows to collect a star and reach home.
- **Teacher prompts**: Which square are we on? What is blocking the way? Can we find a shorter route?
- **Work product**: a grid maze with arrow instructions.

## Chapter 7: Events and Buttons
- **Key concepts**: event, when, button, click, tap, signal, action.
- **Practice activities**: make "when I clap, you jump" games; use cards for button actions; design a toy that responds to a button.
- **Teacher prompts**: What starts the action? What happens when the button is pressed? Does it happen before or after the signal?
- **Work product**: a simple event card: "When ___ happens, ___ happens."

## Chapter 8: Make a Tiny Project
- **Key concepts**: plan, create, test, improve, share, explain.
- **Practice activities**: make a dance algorithm, a grid adventure, a button story, or a repeating pattern artwork; test with a partner.
- **Teacher prompts**: What do you want it to do? Which part repeats? What did you improve?
- **Work product**: a finished unplugged or block-based mini project with one debug note.

## Capstone
Build a classroom robot adventure. Children choose a start and finish on a grid, add obstacles, write arrow cards, include one loop and one if/then rule, test the route with a partner, debug it, and share the final path.

## Assessment Ideas
- Observe whether children can order steps and use first, next, and last correctly.
- Use a movement path to check clear instructions, grid direction, and debugging.
- Ask children to identify a repeated pattern and turn it into a simple loop.
- Review the capstone for a working route, one fix after testing, and a child-friendly explanation.
